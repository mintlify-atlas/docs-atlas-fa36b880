async-profiler is a production-safe, low-overhead sampling profiler for Java. It combines `perf_events` with HotSpot-specific APIs to capture accurate stack traces across Java, native, JVM, and kernel frames — without the safepoint bias that affects most traditional Java profilers.

## Get up and running

<Steps>
  <Step title="Download async-profiler">
    Grab the latest release for your platform from [GitHub Releases](https://github.com/async-profiler/async-profiler/releases/tag/v4.3).

    ```bash
    # Linux x64
    tar -xzf async-profiler-4.3-linux-x64.tar.gz
    cd async-profiler-4.3-linux-x64
    ```
  </Step>
  <Step title="Find your Java process">
    Use `jps` to list running Java processes, or `pgrep`:

    ```bash
    jps
    # 8983 MyApplication
    ```
  </Step>
  <Step title="Profile for 30 seconds">
    Run `asprof` with a duration and output file:

    ```bash
    ./bin/asprof -d 30 -f flamegraph.html 8983
    ```
  </Step>
  <Step title="Open the flame graph">
    Open `flamegraph.html` in any browser to explore an interactive flame graph of your application's CPU usage.
  </Step>
</Steps>

## Output formats

async-profiler can write results as interactive **flame graphs** (HTML), **JFR** recordings (for JDK Mission Control and IntelliJ IDEA), **pprof**, **OTLP**, and more. Use `jfrconv` to convert between formats after the fact.
