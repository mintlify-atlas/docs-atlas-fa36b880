JFR recordings produced by async-profiler can be visualized with several tools. Because the JFR format is an open standard supported by the JDK, async-profiler recordings are compatible with the broader JFR ecosystem alongside async-profiler's own built-in converter.

## Why use JFR format

Recording in JFR format offers several advantages over writing a flame graph directly:

- **Multi-event profiling**: a single recording can capture CPU, allocation, wall-clock, and lock events simultaneously.
- **Deferred visualization**: you can decide which events to visualize after profiling, without re-running the target process.
- **Tooling compatibility**: JFR files work with JMC, IntelliJ IDEA, the `jfr` CLI, and any other JFR-aware tool.
- **Conversion flexibility**: use `jfrconv` to produce flame graphs, heatmaps, pprof, or OTLP from the same recording.

```bash
asprof -d 30 -o jfr -f profile.jfr 8983
```

---

## Built-in converter: jfrconv

async-profiler ships a `jfrconv` utility that converts JFR files into flame graphs, heatmaps, collapsed stacks, pprof, and OTLP. It is the fastest path from a JFR recording to a visual output.

```bash
# Generate a CPU flame graph from a JFR file
jfrconv --cpu profile.jfr

# Generate a heatmap
jfrconv --cpu -o heatmap profile.jfr heatmap.html
```

See [Converter usage](/output/converter-usage) for the full reference.

## JDK Mission Control (JMC)

[JDK Mission Control](https://www.oracle.com/java/technologies/jdk-mission-control.html) is a GUI tool originally designed for the JDK Flight Recorder. async-profiler recordings are fully compatible with JMC.

<Steps>
  <Step title="Download JMC">
    Download JDK Mission Control from the [Oracle JMC downloads page](https://www.oracle.com/java/technologies/jdk-mission-control.html) or from your JDK distribution's tooling package.
  </Step>
  <Step title="Open the recording">
    Launch JMC, then open your `.jfr` file via **File → Open File**.
  </Step>
  <Step title="Navigate to the relevant sections">
    The most useful sections for async-profiler recordings are:

    - **Java Application → Method Profiling**: CPU hot methods and call stacks.
    - **Java Application → Memory**: allocation summaries and live object counts.
    - **Java Application → Lock Instances**: Java monitor contention.
    - **JVM Internals → TLAB Allocations**: per-TLAB allocation breakdown.
  </Step>
</Steps>

<Note>
Some JMC tabs may be empty or show no data for async-profiler recordings. This is expected — async-profiler emits only the events it supports, and JMC tabs that rely on JVM-internal events not captured by async-profiler will have nothing to display.
</Note>

## IntelliJ IDEA

IntelliJ IDEA provides two paths for viewing JFR files, depending on your edition.

<Tabs>
  <Tab title="IntelliJ IDEA Ultimate">
    IntelliJ IDEA Ultimate includes a built-in JFR viewer that works directly with async-profiler recordings. Open a `.jfr` file from the **File** menu or the **Profiler** tool window, or profile a process directly from within the IDE.
  </Tab>
  <Tab title="Community Edition">
    Install the open-source [Java JFR Profiler plugin](https://plugins.jetbrains.com/plugin/20937-java-jfr-profiler) from the JetBrains Marketplace. The plugin allows you to:

    - Profile Java applications directly using JFR and async-profiler as the backend.
    - Open JFR files captured outside the IDE for offline analysis.
  </Tab>
</Tabs>

## jfr command line tool

JDK distributions include the `jfr` CLI utility for filtering, summarizing, and converting JFR recordings to human-readable output.

```bash
# Print a summary of all events in the recording
jfr summary profile.jfr

# Print details of a specific event type
jfr print --events jdk.ExecutionSample profile.jfr

# Filter events by time range
jfr print --from "12:00:00" --to "12:05:00" profile.jfr
```

Refer to the [official jfr documentation](https://docs.oracle.com/en/java/javase/21/docs/specs/man/jfr.html) for the full list of subcommands and options.

## JFR-specific profiler options

When recording in JFR format, async-profiler supports additional options that control the recording behavior:

| Option | Description |
|---|---|
| `--chunksize SIZE` | Maximum size of a single JFR chunk before rotation |
| `--chunktime DURATION` | Maximum duration of a single JFR chunk before rotation |
| `--jfropts OPTIONS` | JFR recording options (comma-separated). Currently supports `mem` on Linux 3.17+: accumulates events in memory instead of flushing to disk |
| `--jfrsync CONFIG` | Synchronize async-profiler events with JVM JFR recordings |
